# 0.0.7 Hardening — refactoring cleanup, scalability, invariant restoration

**Version:** 0.0.7
**Date:** 2026-07-05
**Status:** approved (build mode)
**Decisions locked:** custom `builder` subagent on `opencode-go/deepseek-v4-pro`
(reasoningEffort: max, maxTokens: 384000); Batch 2 full scope (tracer,
PedagogyEngine, RAG hybrid, multi-provider); specs-first workflow.

## Context

Eight `explore` subagents audited the codebase for the 0.0.7 release. Result:
Python suite green (289/289), frontend dev works, F1-F7 fixes verified correct.
One BLOCKING issue (`SkillsTree.tsx` typecheck → `build.sh` cannot produce the
wheel) plus a set of invariant drifts and dead code. This spec restores
invariants and wires the implemented-but-unwired features.

## Execution model

- One `builder` subagent per atomic task, launched in parallel by batch.
- Tasks within a batch touch NON-OVERLAPPING files.
- Each builder: edit + run its scoped verification command; report diffs.
- Coordinator: after each batch, `COGNITS_DISABLE_RAG=1 uv run python -m pytest -q`
  + `cd frontend && bun run typecheck && bun run build` + `scripts/build.sh`.
- Commit after each logical unit (AGENTS.md workflow).

## Prerequisite — T0: builder subagent

**Files:** `.opencode/agents/builder.md` (created)
**Model:** `opencode-go/deepseek-v4-pro`, `reasoningEffort: max`,
`maxTokens: 384000`, `steps: 200`, `mode: subagent`.
**Fallback** (if `task(subagent_type="builder")` is rejected): inline override
of `general` in `opencode.jsonc` with the same model/options, launch via
`subagent_type: "general"`.
**Verify (after opencode restart):** spawn one builder with a trivial task
(`echo ok` + read a file) → confirm it self-reports
`opencode-go/deepseek-v4-pro` and that `reasoningEffort: max` does not conflict
with tool use. If it conflicts, drop to `reasoningEffort: high` or remove it.
